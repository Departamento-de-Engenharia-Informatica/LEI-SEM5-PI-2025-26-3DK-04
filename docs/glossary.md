# Glossary of Terms

| Term                           | Type                    | Description                                                                                                                                                                      |
|--------------------------------|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Vessel**                     | Aggregate Root (Entity) | Represents a ship registered under the port’s management. Identified by its IMO number and name. Each vessel has one type, multiple owners, a crew, and cargo.                   |
| **Owner**                      | Entity                  | Represents an individual or organization owning one or more vessels. Includes identification and name details.                                                                   |
| **Cargo**                      | Value Object            | Describes the type and quantity of cargo associated with a vessel. Used to record transported goods.                                                                             |
| **Crew**                       | Entity                  | Represents the group of personnel assigned to a vessel. Each crew contains multiple crew members.                                                                                |
| **CrewMember**                 | Entity                  | Represents an individual working aboard a vessel. Includes name, nationality, and date of birth.                                                                                 |
| **VesselType**                 | Entity                  | Defines the classification of a vessel based on its characteristics (capacity, structure, and description). Used to associate compatible docks and vessel configurations.        |
| **VesselVisitNotification**    | Aggregate Root (Entity) | Represents a formal notification submitted by a shipping agent for a vessel visit. Includes decision details, rejection reason, and cargo manifests.                             |
| **NotificationState**          | Enum                    | Represents the current state of a vessel visit notification (e.g., InProgress, Submitted, Approved, Rejected).                                                                   |
| **ManifestCargo**              | Value Object            | Represents a cargo entry in a loading or unloading manifest, detailing type and quantity.                                                                                        |
| **LoadingCargoManifest**       | Value Object            | Lists all cargo being loaded onto a vessel during a visit. Each entry references one or more ManifestCargo records.                                                              |
| **UnloadingCargoManifest**     | Value Object            | Lists all cargo being unloaded from a vessel during a visit. Each entry references one or more ManifestCargo records.                                                            |
| **ShippingAgentOrganization**  | Aggregate Root (Entity) | Represents a registered maritime agency responsible for managing vessels and submitting visit notifications. Identified by legal name and tax number.                            |
| **Address**                    | Value Object            | Represents the physical address of an organization, including street, city, postal code, and country.                                                                            |
| **OrganizationRepresentative** | Entity                  | Represents an individual authorized to act on behalf of a ShippingAgentOrganization. Can submit vessel visit notifications. Includes name, nationality, email, and phone number. |
| **Dock**                       | Aggregate Root (Entity) | Represents a physical docking location where vessels are moored for loading, unloading, or maintenance. Includes dimensions and draft limits.                                    |
| **StorageArea**                | Entity                  | Represents a designated area within the port for cargo storage. Includes type, capacity, and occupancy details.                                                                  |
| **Location**                   | Value Object            | Describes the physical or geographical position of a port asset (e.g., dock or storage area), including coordinates and description.                                             |
| **StaffMember**                | Aggregate Root (Entity) | Represents a port employee or officer responsible for operational tasks. Includes personal details, contact information, and work schedule.                                      |
| **MemberStatus**               | Enum                    | Defines the operational or employment status of a staff member (e.g., Active, Inactive, OnLeave, Retired).                                                                       |
| **Qualification**              | Aggregate Root (Entity) | Represents a professional certification or license required to perform specific operational duties in the port.                                                                  |
| **PhysicalResource**           | Aggregate Root (Entity) | Represents tangible equipment or tools used in port operations (e.g., cranes, forklifts). Includes operational capacity and setup requirements.                                  |
| **ResourceStatus**             | Enum                    | Defines the operational condition of a physical resource (e.g., Available, InUse, UnderMaintenance, OutOfService).                                                               |
