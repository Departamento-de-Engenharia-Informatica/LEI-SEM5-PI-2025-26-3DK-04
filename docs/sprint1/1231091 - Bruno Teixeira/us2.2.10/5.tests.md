## 5. Tests

### 5.1. Unit Tests
- **Test Search Own Notifications:**
    - Representative searches without filters.
    - System should return all notifications submitted by that representative.
    - Verify results are ordered by CreatedAt descending.

- **Test Search Organization Notifications:**
    - Representative from organization searches with OrganizationId filter.
    - System should return all notifications submitted by any representative from that organization.
    - Verify representative can see notifications from other representatives in same organization.

- **Test Filter by Vessel:**
    - Search with VesselId filter.
    - System should return only notifications for that specific vessel.

- **Test Filter by Status:**
    - Search with Status filter (Pending, Approved, or Rejected).
    - System should return only notifications with matching status.

- **Test Filter by Representative:**
    - Search with RepresentativeId filter.
    - System should return only notifications submitted by that specific representative.

- **Test Filter by Date Range:**
    - Search with StartDate and EndDate filters.
    - System should return only notifications created within that time period (inclusive).

- **Test Combined Filters:**
    - Search with multiple filters (e.g., Status=Approved AND StartDate/EndDate).
    - System should return notifications matching all filter criteria.

- **Test Empty Results:**
    - Search with filters that match no notifications.
    - System should return empty list without errors.

- **Test Invalid Status Filter:**
    - Attempt to search with invalid status value.
    - System should reject and return validation error.

- **Test Organization Isolation:**
    - Representative from Organization A searches.
    - Verify results do NOT include notifications from Organization B.


### 5.2. Functional Tests
- **Test View Own Notifications via UI:**
    - Representative logs in with Shipping Agent Representative credentials.
    - Representative navigates to "View Notifications".
    - System displays all notifications submitted by the representative.
    - Verify each notification shows vessel, status, submission date.

- **Test View Organization Notifications:**
    - Representative views notifications from their organization.
    - System displays notifications from all representatives in the organization.
    - Verify representative can see who submitted each notification.

- **Test Search by Vessel:**
    - Representative enters vessel name or ID in search.
    - System filters and displays only notifications for that vessel.

- **Test Filter by Status:**
    - Representative selects status filter (e.g., "Approved").
    - System displays only approved notifications.
    - Verify approved notifications show dock assignment.

- **Test Filter by Date Range:**
    - Representative selects start and end dates.
    - System displays only notifications created within that period.

- **Test Combined Filters:**
    - Representative applies multiple filters (vessel + status + date range).
    - System displays notifications matching all criteria.

- **Test Sort Order:**
    - Verify notifications are displayed with most recent first.

- **Test Notification Details:**
    - Verify each notification displays:
        - Vessel information
        - Current status
        - Representative who submitted it
        - Submission date/time
        - Dock assignment (if approved)
        - Rejection reason (if rejected)

- **Test Organization Isolation:**
    - Representative from one organization should not see notifications from other organizations.
