## 2. Analysis

### 2.1. User Story Overview

**ID:** US_2.2.1  
**Title:** Create and Update Vessel Types

**Description:**  
As a Port Authority Officer, I want to create and update vessel types so that vessels can be classified consistently and their operational constraints are properly defined.
Each vessel type includes identifiers, names, description, capacity, and operational constraints (e.g.: maximum number of rows, bays, and tiers).

---

### 2.2. Business Rules

1. **User Authentication:**
    - Only authenticated users with the role *Port Authority Officer* can create or update vessel types.
2. **Data Consistency:**
    - Each vessel type must include a unique identifier.
    - Each vessel type must have a unique name.
    - Vessel type descriptions must be unique.
3. **Notification:**
    - Should notify the user upon successful or unsuccessful creation or update of a vessel type.
